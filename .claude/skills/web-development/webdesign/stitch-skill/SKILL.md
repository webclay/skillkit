# Stitch Design Taste: Semantic Design System Skill

## Summary

Generates `DESIGN.md` files for Google Stitch, translating premium UI principles into agent-readable semantic design language. The framework enforces anti-generic standards through strict rules on typography, color calibration, asymmetric layouts, and micro-motion.

## Core Framework

**Visual Atmosphere Definition:**
The system operates on three spectrums - Density (1-10), Variance (1-10), and Motion (1-10) - with baseline values at Variance 8, Motion 6, and Density 4, adjusted per project vibe.

**Color Discipline:**
Limited to one accent color with saturation below 80%. "The 'AI Purple/Blue Neon' aesthetic is strictly BANNED - no purple button glows, no neon gradients." Neutral bases using Zinc or Slate palettes paired with high-contrast singular accents.

**Typography Architecture:**
Inter font is prohibited for premium contexts. Recommended alternatives include Geist, Outfit, Cabinet Grotesk, or Satoshi. Generic serif fonts are banned in dashboards; modern serifs (Fraunces, Editorial New) are acceptable only in editorial applications.

**Layout Governance:**
"No overlapping elements - every element occupies its own clear spatial zone. No absolute-positioned content stacking." Centered hero sections are banned for high-variance projects. Multi-column card grids must use asymmetric arrangements instead of uniform three-column layouts.

**Motion Standards:**
Spring physics defaults to stiffness 100, damping 20. All animations use transform and opacity exclusively - never animate positional or dimensional properties. Perpetual micro-interactions provide visual feedback on active states.

## Anti-Pattern Enforcement

Explicitly bans: emojis, pure black (#000000), neon glows, custom cursors, generic placeholder names, AI copywriting clichés, and filler UI text like "Scroll to explore."

This design system operates as the single source of truth for Stitch screen generation, ensuring consistency and preventing generic outputs.
